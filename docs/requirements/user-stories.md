# 用户故事 (User Stories)

## 读者角色定义

| 角色 | 标识 | 背景 | 目标 |
|------|------|------|------|
| **入门开发者** | US-BEGINNER | 刚接触 AI 编程，基本编程能力 OK | 快速上手 OpenCode，在日常开发中用起来 |
| **效率开发者** | US-POWER | 已用 AI 工具(Copilot/Cursor)，想升级 | 掌握 Agent 编排，提升 2x+ 效率 |
| **技术负责人** | US-LEAD | 团队技术决策者，关注标准化 | 建立团队级 Harness Engineering 体系 |
| **Skill 作者** | US-SKILL | 有一定 AI 使用经验，想扩展能力 | 掌握 Skill 开发方法，产出高质量 Skill |
| **工程经理** | US-MANAGER | 评估团队工具选型 | 判断 OpenCode 的投资回报率 |
| **需求分析师/产品经理** | US-ANALYST | 需求分析、产品规划经验 | 验证需求覆盖完整性、评估内容价值主张 |
| **系统架构师/技术顾问** | US-SYSA | 5年以上架构经验，负责技术决策 | 评估OpenCode的技术可行性、架构集成与安全合规 |
| **后端开发者/API工程师** | US-BACKEND | 熟悉REST/GraphQL/微服务/数据库 | 将AI Agent嵌入后端开发工作流、MCP服务端集成 |
| **前端开发者/UI工程师** | US-FRONTEND | 熟悉React/Vue/Angular、组件化开发 | 将Agent编排应用到前端场景、类比理解Skill系统 |
| **文档UX专家** | US-UX | 信息架构/开发者文档经验 | 确保文档可读性、Mermaid规范、移动端/无障碍体验 |
| **技术审校/QA编辑** | US-QA | 测试或技术写作背景 | 建立质量门禁、验证代码示例可运行性、术语一致性 |
| **安全工程师/架构师** | US-SECURITY | 安全工程/合规/威胁建模 | 建立OpenCode安全基线、评估企业级合规 |
| **安全研究人员/红队成员** | US-REDTEAM | 渗透测试/安全研究 | 评估AI Agent攻击面、利用Agent自动化安全测试 |

## 入门开发者 (BEGINNER)

### US-BEGINNER-01：快速了解核心价值
> **作为** 一个刚接触 OpenCode 的开发者，
> **我想要** 在 10 分钟内理解它能做什么、和 Cursor/Claude Code 有何不同，
> **以便** 判断是否值得花时间学习。

**验收标准：**
- [ ] Ch1 包含直观的对比表（OpenCode vs Cursor vs Claude Code vs Copilot）
- [ ] 有 3 个核心优势的简明解释
- [ ] 包含 Roadmap 建议：哪些场景 OpenCode 最强

**优先级：** P0 | **工作量：** S

### US-BEGINNER-02：5 分钟完成安装
> **作为** 一个入门开发者，
> **我想要** 看到 Mac/Windows/Linux 三种平台的安装步骤，
> **以便** 快速完成环境搭建。

**验收标准：**
- [ ] 3 种平台的安装步骤（官方脚本 / npm / Homebrew / Scoop）
- [ ] Provider 配置方式（API Key / OpenCode Zen / GitHub Copilot）
- [ ] 验证安装的 check 命令

**优先级：** P0 | **工作量：** S

### US-BEGINNER-03：第一个成功的尝试
> **作为** 一个入门开发者，
> **我想要** 一个"Hello World"级别的上手教程，
> **以便** 建立初次成功的信心。

**验收标准：**
- [ ] 3 步以内完成第一个有用操作
- [ ] 包含 Plan/Build 模式的概念和切换
- [ ] 包含 /undo 撤销操作

**优先级：** P0 | **工作量：** S

### US-BEGINNER-04：理解 6 个核心概念
> **作为** 一个入门开发者，
> **我想要** 用类比的方式理解 Agent/Skill/Command/Plugin/Team/MCP，
> **以便** 知道什么时候用什么工具。

**验收标准：**
- [ ] 每个概念有操作系统类比（如 Agent = 进程，Skill = 驱动程序）
- [ ] 概念关系图（展示 6 个概念如何联动）

### US-BEGINNER-05：从"对话"到"工程"的认知转变
> **作为** 一个入门开发者，
> **我想要** 理解"Harness Engineer"和普通 AI 用户的区别，
> **以便** 建立工程化的思维方式。

**验收标准：**
- [ ] 清晰的"三个阶段"演进图（代码补全→对话编程→工程流水线）
- [ ] 3 个具体的思维转变示例
- [ ] Harness Engineer 的核心能力清单

**优先级：** P1 | **工作量：** S
