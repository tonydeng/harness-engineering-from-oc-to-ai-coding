## 二、逆向检查：什么可能误导读者？

| 潜在误导 | 具体问题 | 影响分析 |
|---------|---------|---------|
| **工作流模式 ≠ 工作流系统** | README 原本描述四种模式（顺序/并行/审核/迭代），文章实际讨论的是 Command/Profile/AGENTS.md。这四者完全不同 | 读者若想了解"顺序执行模式"会直接跳转到此文章，通读后找不到相关内容，产生困惑 |
| **"工程实现"暗示已存在** | 验证护栏文章中反复声明质量门禁和自动修复是"架构建议"而非内置功能，但 README 原文使用了"工程实现"一词，暗示这些已经是 OpenCode 的稳定功能 | 读者可能误以为 OpenCode 原生具备质量门禁能力，实际需要通过第三方工具自行搭建 |
| **"任务分配策略"具体程度** | 排版上读写者可能期待一个类似"分配策略分类"的章节，但文章仅通过选择决策树和 OMO 扩展间接提供 | 读者逐条核对学习目标时可能不知道自己该找什么 |

**以上已通过修正解决前两项问题。**
