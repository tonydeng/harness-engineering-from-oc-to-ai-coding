### 概念框架层面的问题（超出本次修正范围）

数据研究报告指出全书存在系统性术语偏差——将行业通用概念和作者构建的分类框架包装为 OpenCode 平台特性。具体包括：
- Workflow 被表述为与 Agent/Skill 并列的一等抽象，但官方无此概念
- "上下文工程"、"约束系统"、"验证护栏体系"均为本书自创分类，非 OpenCode 官方子系统名称
- 约束层级（全局/会话/任务）与官方层级（远程/全局/项目）不匹配

这些问题涉及全书的概念框架定位，建议在后续章节迭代中统一考量。
