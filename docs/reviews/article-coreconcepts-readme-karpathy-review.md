## Karpathy 工程现实主义视角补充分析

1. **三个核心抽象（Agent/Skill/Workflow）的分解是合理的**：从系统架构角度，将 AI 编程能力分为执行单元、知识包、编排模式三层，是干净的工程抽象。术语表中 Agent = Model + Tools + Skills + Memory 的四要素定义更是直击要害。

2. **三个支撑子系统（上下文工程/约束系统/验证护栏）的定位准确**：
   - 上下文工程 → 输入阶段（Agent 能"看到"什么），对应实际工程中的 Token 预算管理和压缩
   - 约束系统 → 执行阶段（Agent "能/不能"做什么），对应实际工程中的权限模型和沙箱
   - 验证护栏 → 输出阶段（结果"对不对"），对应实际工程中的 CI 质量门禁和自动化测试
   
   这个输入→执行→输出的分段模型在工程上是正确的。

3. **上下文工程的双重身份问题**（既是历史阶段又是当前子系统）在书中已通过时序演进 + 当前实践的框架做了合理交代，不构成事实错误。
