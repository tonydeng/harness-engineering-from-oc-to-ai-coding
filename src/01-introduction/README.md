# 第1章：简介 — 从 AI 聊天到工程体系

> 本章为你建立 Harness Engineering 的思想坐标——理解什么是"驾驭工程"、为什么选择 OpenCode，以及它在当前 AI 编程工具生态中的位置。

## 章节概述

第 1 章是全书的认知基础。我们首先定义 **Harness Engineer（驾驭工程师）** 的核心思想——从"跟 AI 聊天写代码"升级到"用工程体系做开发"。然后深入分析为什么 OpenCode 是承载这一理念的最佳平台，并通过工具生态对比帮助你建立全面的技术选型视角。最后，针对国内开发者关心的国产模型适配问题，给出具体的集成方案。
