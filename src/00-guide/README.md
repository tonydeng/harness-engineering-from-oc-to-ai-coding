# Ch0 读者导航 — 本书适合你吗？

> 本书不是"从零学编程"教程，而是帮助你从"用 AI 聊天写代码"升级到"用 AI 工程流水线做开发"的实践指南。

---

## 章节概述

第 0 章是全书的入口，帮助你快速判断这本书是否适合你，以及如何最高效地阅读。根据你的角色和技术背景，选择不同的阅读路径，可以让你在最短时间内获取最有价值的内容。

本章包含以下内容：

- **[多角色阅读路径](reading-paths.md)** — 针对 13 种读者角色（初学者、团队负责人、安全工程师、Skill 开发者等）提供定制化的章节跳转建议，以及对应的阅读时间估算。
- **[如何使用本书](how-to-read.md)** — 两种阅读模式（逐章精读 vs 按需跳跃）的对比说明，以及最大化学习收益的实操建议。

---

## 我是谁？我应该读哪些章节？

| 你的画像 | 建议阅读路径 | 估计用时 |
|----------|-------------|----------|
| **AI 编程新手** — 用过 Copilot/Claude Code，想理解 Agent 编排 | Ch1 → Ch2 → Ch3 → Ch4 → Ch6 | 4-5 小时 |
| **OpenCode 用户** — 已有基础，想深入学习 Skill 开发和 Workflow 设计 | Ch2 → Ch4 → Ch5 → Ch6 | 3-4 小时 |
| **团队技术负责人** — 评估或导入 OpenCode 作为团队工具 | Ch1 → Ch3 → Ch4 → Ch7 | 3-4 小时 |
| **安全/架构师** — 关注安全模型和可观测性 | Ch3 §3.2 → Ch6 → Ch7 | 2-3 小时 |
| **Skill 开发者** — 想创建和发布自定义 Skill | Ch2 §2.2 → Ch5 → Ch6 §6.2 | 3-4 小时 |
| **案例学习者** — 通过真实项目学习 | Ch7 → 按需回溯前序章节 | 2-3 小时 |

> 所有章节都设计为可独立阅读，但建议至少读过 Ch1 和 Ch2 以掌握基础概念。

## 这本书不讲什么

| 内容 | 为什么跳过 | 替代资源 |
|------|-----------|---------|
| 编程基础（语法、变量、函数） | 假设读者已有开发经验 | — |
| OpenCode 的 Rust 内部实现 | 只关注用户层面配置 | OpenCode 官方文档 |
| 每个框架/AWS/GCP的完整教程 | 聚焦 AI 编程工作流本身 | 各框架官方指南 |
| 大模型训练或微调 | 本书是工程实践，不是 ML 教程 | Hugging Face / 各模型文档 |

## 需要哪些前置知识？

- **基础**: 至少熟悉一种编程语言（TypeScript/Python/Go 等）
- **工具**: 基本的命令行和 Git 使用经验
- **AI**: 使用过至少一种 AI 编程助手（Copilot / Claude Code / Cursor 等）
- **可选**: 了解 Agent / LLM / MCP 的基本概念会加速理解，但不是必须的

## 如何从本书获得最大收益？

1. **打开两个窗口** — 一个看本书，一个打开你的编辑器跟着操作
2. **先理解概念，再复制配置** — 每章的示例都是可运行的，但请确保理解它为什么这样设计
3. **跳过不匹配的部分** — 如果你只用 VS Code，可以跳过 TUI 相关章节
4. **遇到问题时先看 FAQ** — 各章节末尾都有常见问题和排查指南
5. **带着你的真实项目来读** — 把示例中的概念映射到你的实际场景
