# 多角色阅读路径

> 不同背景的读者，从本书获取价值的路径各不相同。本文帮助你找到属于自己的那条路。

## 文章概述

一本涵盖 8 章 46 篇文章的技术书，从头读到尾并不是最高效的选择。本书设计了 13 种读者角色分类，每种角色对应不同的阅读路径。你可以根据自己的技术背景、职业角色和学习目标，跳过不相关的章节，直达最有价值的内容。

阅读路径不是简单的章节列表。每条路径都标注了预计阅读时间、建议的阅读顺序，以及哪些小节可以跳过。对于团队负责人和评估者，路径中还包含了对 Ch3 环境搭建和 Ch7 案例研究的定向指引。无论你是第一次接触 **Agent（智能体）** 编排的新手，还是已有 OpenCode 使用经验的老手，都能找到适合自己的路线。

## 内容要点

1. **13 种读者角色分类** — 基于 45 个用户故事提炼出的角色矩阵，涵盖初学者、技术负责人、安全工程师、Skill 开发者等，每位读者都能在其中找到自己的定位。

2. **AI 编程新手路径** — 从 Ch1 开始建立 Harness Engineering 的概念框架，经 Ch2 核心概念打底，进入 Ch3 环境搭建和 Ch4 工作流实战，最后了解 Ch6 高级话题。适合用过 Copilot 或 Claude Code 但想升级到 Agent 编排的开发者。

3. **OpenCode 用户进阶路径** — 跳过环境搭建章节，直接从 Ch2 深入理解 Agent、Skill、Workflow 的抽象模型，然后进入 Ch4 和 Ch5 学习如何设计和发布自定义 Skill。适合已有 OpenCode 基础的用户。

4. **团队技术负责人路径** — 聚焦评估和导入 OpenCode 所需的信息：从 Ch1 理解 Harness Engineering 的价值主张，到 Ch3 评估多环境部署方案，再到 Ch4 团队工作流设计，最后通过 Ch7 案例研究验证可行性。

5. **安全工程师 / 架构师路径** — 重点关注 Ch3 的国产模型集成与安全配置、Ch6 的安全模型和可观测性，以及 Ch7 的安全审计案例。适合需要评估 AI 编程工具安全合规性的读者。

6. **Skill 开发者路径** — 以 Ch2 的 Skill 抽象模型为起点，深入 Ch5 的 Skill 开发与最佳实践，然后探索 Ch6 的 MCP 桥接和插件模式。适合计划创建和发布自定义 Skill 的开发者。

7. **案例学习者路径** — 适合喜欢从真实项目入手的读者。从 Ch7 的案例研究开始，遇到不理解的概念时回溯到前序章节查找。预计 2-3 小时完成。

8. **各路径时间估算与阅读节奏建议** — 提供每条路径的预估阅读时间（2-5 小时不等），以及如何安排阅读节奏的建议。也说明了哪些情况下适合切换路径或混合阅读。
