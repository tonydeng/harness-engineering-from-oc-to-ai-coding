# CLAUDE.md 约定系统

> 如果说 AGENTS.md 是项目的"宪法"，CLAUDE.md 就是用户的"行政令"。一个定义根本规则，一个下达具体指令，两者配合构成完整的指令覆盖体系。

## 文章概述

在 AI 编程流水线中，指令的来源是分层的。AGENTS.md 定义了项目的根本规则和架构决策——它是编写者视角的约束。CLAUDE.md 则代表了使用者的直接指令——用户告诉 Agent "当前这个任务，我希望你这样做"。两者的核心区别：AGENTS.md 是写给所有 Agent 的通用规范，CLAUDE.md 是针对当前项目和任务的特定指令。

本文首先阐述 CLAUDE.md 在指令体系中的定位——用户指令覆盖层。然后对比 CLAUDE.md 与 AGENTS.md 的职责划分：AGENTS.md 负责架构级约束、编码规范、项目结构，CLAUDE.md 负责具体任务指令、开发偏好、临时策略。接着介绍指令覆盖策略的三层结构：全局指令（`~/.claude/CLAUDE.md`）、项目指令（项目根目录 `CLAUDE.md`）、子目录指令（通过 `@include` 引入）。最后深入 `@include` 指令系统——文件包含、指令嵌套、优先级和合并规则，并给出团队级指令管理的最佳实践。

## 内容要点

1. **CLAUDE.md 的作用** — 用户指令覆盖层的设计思想：用户可以随时覆盖 Agent 的行为。与 AGENTS.md 的职责划分：AGENTS.md = 项目的"宪法"（架构规则、编码规范）、CLAUDE.md = 用户的"行政令"（具体怎么做、偏好什么工具）。

2. **指令覆盖策略** — 三层覆盖结构：全局指令（`~/.claude/CLAUDE.md`，所有项目通用）、项目指令（项目根目录 `CLAUDE.md`，项目特定）、子目录指令（`@include` 引入，目录级特定）。每层的覆盖优先级和合并规则。

3. **@include 指令系统** — 包含外部文件的语法（`@include path/to/file.md`），指令嵌套（被包含的文件自身也可以 @include），指令的优先级和合并规则（就近优先、显式覆盖隐式、后加载覆盖先加载）。指令冲突检测和解决方案。

4. **最佳实践** — 什么内容放在 AGENTS.md 还是 CLAUDE.md（用 AGENTS.md 定规则，用 CLAUDE.md 给指令）。团队级指令管理的建议（统一模板、定期审查、变更记录）。开发流程中的指令更新策略（任务开始前检查和更新 CLAUDE.md）。
