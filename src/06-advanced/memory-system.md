# 记忆系统设计

> 缓存让 Agent 记住"系统知道的"，记忆让 Agent 记住"自己经历过的"。从 Memdir 架构到 Auto-Dream 机制，构建 Agent 的长期记忆系统。

## 文章概述

缓存是被动的——它存储的是系统预设的内容（系统指令、工具定义、项目知识）。记忆是主动的——它记录的是 Agent 在执行任务过程中的上下文、决策和发现。这是两者的本质区别。记忆系统解决的问题是：当 Agent 从一个 Session 进入下一个 Session 时，如何不忘记之前做过什么、发现过什么、决定了什么。

本文首先澄清"记忆 vs 缓存"的概念差异。然后深入 Memdir 架构——这是记忆系统的文件组织方式，包括目录结构设计、记忆文件的格式和索引、读写策略。接着讲解 Auto-Dream 机制——Agent 自动生成记忆摘要、评估信息重要度、淘汰低价值记忆的核心过程。然后分析 Compaction 与记忆系统的配合：Compaction 保留重要决策和上下文，记忆为 Compaction 提供优先级参考。最后讨论记忆系统的安全考虑——敏感信息保护和多项目隔离。

## 内容要点

1. **记忆 vs 缓存** — 记忆是被动的（Agent"记得"什么），缓存是被动的（系统"存了"什么）。记忆系统解决的核心问题：跨 Session 上下文保持。记忆的三个层次：短期记忆（当前 Session）、中期记忆（相关 Session）、长期记忆（项目级知识）。

2. **Memdir 架构** — 目录结构设计原则（按时间/项目/主题组织），记忆文件的格式（结构化元数据 + 自然语言摘要），索引机制（快速检索和相关性排序），读写策略（写入时机、读取优先级、合并策略）。

3. **Auto-Dream 机制** — 自动生成记忆摘要的工作原理（Agent 在 Session 结束时自动总结），记忆重要度评分（基于任务类型、决策影响、用户反馈），记忆自动淘汰策略（空间不足时按重要度最低优先淘汰）。

4. **Compaction 与记忆的配合** — Compaction 在保留重要决策和上下文时如何参考记忆系统的优先级排序，记忆作为 Compaction 的输入来源（优先压缩不重要信息）。

5. **记忆系统的安全考虑** — 敏感信息保护（不在记忆文件中存储 API Key、密码等），多项目隔离（每个项目的记忆独立存储、独立管理），记忆的导出和备份策略。
