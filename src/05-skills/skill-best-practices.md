# Skill 最佳实践

> 从真实项目中提炼的 Skill 设计原则、反模式清单和调试方法，避免踩坑，写出高质量的 Skill。

## 文章概述

创建 Skill 很容易，但写出高质量的 Skill 需要经验和判断力。本文汇集了来自多个真实项目的实践总结：什么样的 Skill 设计是好的？哪些"看起来不错"的做法其实是反模式？当 Skill 不按预期工作时应该从哪里入手排查？

本文从 6 条核心设计原则出发，剖析 10 种以上常见反模式及其正确做法，提供可操作的 8 步调试清单，并讨论 Skill 在 Team Mode 中的集成策略——特别是 target_agent 与类别路由的协同工作原理。这些经验不仅适用于个人开发者，对团队层面的 Skill 治理同样有参考价值。

## 内容要点

1. **Skill 设计 6 条核心原则** — 单一职责（一个 Skill 只解决一个领域问题）、可组合（通过 Agent 编排串联）、精准触发（description 精确到不用看正文）、最小权限（仅开放必需工具）、可测试（明确输入输出和验证方式）、版本声明（语义化版本和兼容性标注）。
2. **10 种以上反模式及纠正方案** — 涵盖超长 Skill、全能 Skill、Agent 与 Skill 混淆、硬编码模型名、缺少工具白名单、忽略作用域限定、测试依赖外部系统、版本缺失、指令模糊、捆绑大型资源等问题，每项均配有错误示例和正确做法。
3. **8 步调试清单** — 从 SKILL.md 格式检查、配置文件验证、路径排查、禁用状态确认、作用域检查、描述匹配验证、覆盖冲突检测到 Agent 类型适配，提供每一步的具体排查命令。
4. **Team Mode 中的 Skill 集成** — target_agent 与类别路由的协同机制，团队成员间 Skill 不可见的设计价值，以及 Overrides 的最佳使用场景。
