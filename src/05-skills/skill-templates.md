# Skill 模板

> 4 个即开即用的 Skill 模板，覆盖调查研究、架构设计、代码审查和敏捷活动四大场景。

## 文章概述

一个好的 Skill 模板不仅仅是代码模板，它是一类问题的设计模式——告诉你在特定场景下应该组织哪些步骤、调用哪些工具、产出什么结果。本文提供 4 个经过实战检验的完整 SKILL.md 模板，读者可以直接复制使用，也可以根据自身需求进行定制。

每个模板都包含设计动机、适用场景、完整 SKILL.md 正文、核心工具链以及定制指南。通过对比 4 个模板的设计差异，读者还能加深对 Skill 方法论本质的理解——模板不是"抄作业"，而是"学思路"。

## 内容要点

1. **模板 1：调查研究 Skill** — 用于技术选型、竞品分析、领域调研等场景，核心能力组合为 websearch + read + mindmap 输出，提供完整的调研流程和输出格式。
2. **模板 2：架构设计 Skill** — 用于系统设计和技术方案产出，核心能力为 uml + architecture + read，配备 ADR（架构决策记录）作为标准输出模板，确保架构设计过程可追溯。
3. **模板 3：代码审查 Skill** — 用于 PR 审查和代码质量检查，核心能力为 read + grep + lsp_diagnostics，附带结构化的审查清单模板，确保每次审查的覆盖度和一致性。
4. **模板 4：敏捷活动 Skill** — 用于站会、回顾和 Sprint 规划等团队活动，核心能力为 read + edit（仅限会议记录写入），包含各类敏捷活动的引导流程和产出规范。
5. **模板的选择与定制** — 提供模板选择决策树和能力组合设计指南，帮助读者根据实际需求定制或混搭模板。
