# Harness Engineering — From OpenCode to AI Coding

> **将 AI 编程从"聊天对话"升级为"工程流水线"**

## 为什么要写这本书？

AI 编程工具正在以惊人的速度进化。从 Copilot 的代码补全，到 Cursor/Claude Code 的对话式编程，再到 OpenCode 的 Agent 编排——但大多数人仍然在用"聊天工具"的方式使用它们。

**Harness Engineer**（驾驭工程师）不是简单地"用 AI 写代码"，而是：

- **工程化**：把 AI 编程当作一条流水线来设计和管理
- **系统化**：将 Agent、Skill、Workflow 组合成可复用的生产力体系
- **可持续**：让 AI 辅助的工程实践可维护、可演进、可传承

## 适用读者

- 已在使用 AI 编程工具、想进一步提升效率的开发者
- 正在评估或导入 OpenCode 的团队技术负责人
- 对 Agent 编排、Skill 开发感兴趣的 AI 工程师

## 如何使用本书

每一章都包含：
- **概念讲解** — 核心思想和原理
- **实战示例** — 可直接运行的配置和代码
- **最佳实践** — 来自真实项目的经验总结

你可以从头读到尾，也可以按需跳转到感兴趣的章节。
